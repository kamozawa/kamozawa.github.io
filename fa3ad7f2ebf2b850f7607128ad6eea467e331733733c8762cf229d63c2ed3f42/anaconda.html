Python環境の導入方法

Pythonを利用するためには，それを動作させるための"環境"が必要になります。 Python本体のバージョン，モジュールのバージョン等の互換性を考えると，どうしても複数の環境が必要となるため， 何か新しくプロジェクトを作成する際は仮想環境と呼ばれるまっさらな環境を作ります。

Python環境としては「venv」と「conda」が有名です。 ここでは，「conda」環境の導入方法を説明します。
Anacondaのインストール

ここでは，比較的管理がしやすいAnaconda（アナコンダ）を用いた仮想環境の作成方法について説明します。

まず，Anacondaの公式サイトにアクセスし， 自分のOSに合ったインストーラをダウンロードしましょう。
その後，インストーラを実行します。
インストーラでは様々なカスタムセットアップができますが，特に変更する必要はありません。 Installation complete, setup was completed successfully. が出ればインストール完了です。

仮想環境の作成

Anacondaが正常にインストールされていれば，Anaconda Promptが入っているはずです。
検索バー等からAnaconda Promptを探し，起動してください（Anaconda Powershell Promptではない方です）。

Anaconda Promptを起動すると，右図のような黒い画面が出てくると思います。
(base)になっていることを確認してください。

以下のコマンドで新たな仮想環境を作成します。 

 conda create -n 環境名 python=x.x

環境名は全角文字が入っていなければ何でもOKです。xには導入したいPythonのバージョンを入力します。
機械学習関連であれば，3.6をお勧めします。
入力すると何だかごちゃごちゃ出てきますが，"y"を押してインストールを進めましょう。
これで仮想環境の作成と，Pythonの導入は完了です。

エディタのインストール

もうすでにエディタはあるよ，という方は飛ばして頂いても結構です。

Pythonのコードを書いたり実行するにあたり，高機能のエディタは必須です。究極メモ帳や秀丸エディタでも書けますが，かなり苦労します。
私はJetBrains社のエディタ信者なので，PyCharmを紹介します。
JetBrains社のエディタは学生ライセンスが存在し，なんと有料版を無償で使用できます！（ここでは説明しません）

公式サイトから インストーラをダウンロードして実行すればOKです。

インストールできたら，起動してみましょう。

起動画面右上あたりにある，「NEW PROJECT」を選択してください。

Pythonのコードを書くので，左メニューから"Pure Python"を選択します。
「Location」では，コード（プロジェクト）を保存するフォルダのパスを指定します。

ここからは結構重要です。
「Python Interpreter」の設定では，プルダウンメニューから"Conda"を選択します。

Condaが選択された時点で，すでに存在する環境を自動検索してくれる場合がありますが，そうでない場合は， 環境のある場所，Pythonのバージョン，実行ファイルの場所を手動で設定する必要があります。
普通は"C:/Users/ユーザ名/Anaconda3/envs"下に環境のファイルがありますので，注意して確認してください。

インタプリタ設定が合わないとコードが実行できないので，しっかりと確認を！

設定が終わったら，「CREATE」を押して準備完了です。
"Make available to all projects"にチェックを入れておくと，ちょっと便利です。
あと，そんなにビビらなくても，インタプリタ設定はいつでも変更できます。大丈夫です。

パッケージの追加インストール方法

コードを書いていると，「○○というモジュールはないぞ！」と怒られることがよくありますね。 そこで，作成した仮想環境への，新たなパッケージのインストール方法を紹介します。

まず，例によってAnaconda Promptを起動しましょう。 起動できたら， 

conda activate 環境名

と打ちます。
すると，(base)環境だった表示が，作成した環境名に変わっていると思います。
その状態で， 

conda install パッケージ名                   または，
conda install -c conda-fore パッケージ名

と入力します。
うまく行けば，何かごちゃごちゃ出てきて[y/n]の選択を迫られるので，"y"を入力してやればインストールできます。

ごちゃごちゃ出てきたのにインストールできなかった場合は，モジュール名を間違えているか，そもそもCondaで配布されていない，ということになります。
モジュール名と，インストール時に入力する名前が異なる場合もあるので，注意してください。
（たとえば，scikit-learnというパッケージは，conda install sklearnと打たないとインストールできません）

Condaで配布されていなくても，GitHubから落としてローカルに配置するという力技も可能なので，インストールに失敗しても諦めないでくださいね。
学内ネットワークでインストールする場合は，プロキシ設定が効いてくる場合もあるので，全然インストールできない！となったら聞いてください。
